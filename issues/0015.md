# 0015 — Jump to top and bottom: g / G and Home / End

| | |
|---|---|
| **Status** | open |
| **Module** | Keys / Pager |
| **Platform** | macOS |
| **First seen** | 2026-06-10 |

## Description

`less`'s most-used navigation after paging is the leap to either end: `g` (or Home) jumps to the first line, `G` (or End) to the last. NeoPager has line and page movement (#0006) but no way to jump straight to the top or bottom of a large buffer.

## Expected behavior

- `g` and Home → scroll to the very top (offset 0); `atTop` true.
- `G` and End → scroll to the bottom (offset `maxOffset`); `atEnd` true, status bar shows `END`.
- Both are no-ops (no repaint) when already at that end.

## Notes

`g`/`G` are printable characters, so they arrive as `.char("g")` / `.char("G")` in the key handler and route to new `PagerState.scrollToTop()` / `scrollToBottom()` methods (clamped, emit `objectWillChange` only on a real move — same pattern as the existing movement methods).

Home/End are escape sequences and need `RawKeyParser` additions: Home = `ESC[H` or `ESC[1~` (and SS3 `ESC O H`), End = `ESC[F` or `ESC[4~` (and SS3 `ESC O F`). Add `.home` / `.end` events and map them to the same scroll-to-top/bottom actions.

Phase-2 caveat (shared with #0013/#0014): `g`/`G` are literal query characters in search-input mode (#0010) — the jump bindings apply only in normal mode.
