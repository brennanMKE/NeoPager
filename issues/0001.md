## Description

The package only depends on swift-argument-parser, has no platform constraint, and produces an executable named `NeoPager`. SwiftTUI must be added and the package configured before any TUI code can be written.

## Expected behavior

`Package.swift` declares `platforms: [.macOS(.v13)]`, adds the SwiftTUI dependency (`https://github.com/rensbreur/SwiftTUI.git`, pinned to branch `main`), threads the product into the target, applies `swiftSettings: [.defaultIsolation(MainActor.self)]`, and declares an executable product named `neopager` (lowercase, the installed command name).

## Notes

Mirror the configuration in `../TerminalDashboard/Package.swift`, which uses exactly this setup (SwiftTUI on branch `main`, macOS 13+, MainActor default isolation, Swift 6 language mode). SwiftTUI has historically not tagged semver releases, hence the branch pin. Verify with `./build.sh clean build`.

## Root cause

Greenfield setup, not a defect: the package was scaffolded with only swift-argument-parser, no platform floor, and a capitalized `NeoPager` executable — none of the TUI prerequisites were in place.

## Fix

Rewrote `Package.swift` to mirror `../TerminalDashboard/Package.swift`: added `platforms: [.macOS(.v13)]`, the SwiftTUI dependency pinned to branch `main`, threaded the `SwiftTUI` product into the `NeoPager` target, set `swiftSettings: [.defaultIsolation(MainActor.self)]`, and declared an executable product `neopager`. The `StrictConcurrency` upcoming-feature flag was dropped because Swift 6 language mode (`swiftLanguageModes: [.v6]`) already implies complete strict concurrency.

## Verification

`./build.sh clean build` — clean build succeeded (SwiftTUI compiled and the `neopager` product linked). `echo "" | swift run neopager` ran the executable, and `Package.resolved` now pins `swifttui` to the `main` branch.

## Files changed

- `Package.swift` — added macOS 13 platform, SwiftTUI dependency + product wiring, `neopager` executable product, simplified `swiftSettings`.
- `build.sh` — `run` action now invokes `swift run neopager` to match the new product name.
- `Package.resolved` — regenerated to pin SwiftTUI (`main`) alongside swift-argument-parser.

## Gotchas

Declaring an explicit `.executable(name: "neopager", ...)` product while the target stays `NeoPager` means the installed/runnable command is `neopager`, not `NeoPager`. Any script or doc that referenced `swift run NeoPager` must switch to `neopager` (build.sh was the only one).
