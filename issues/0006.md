## Description

`PagerState` is the observable scroll model: ↑/↓ move the offset one line, ⌥↑/⌥↓ move one full viewport, and movement clamps at both ends. Crucially — the original sin of `more` — reaching the bottom must not exit the pager; the user keeps full freedom to scroll back up until they press Esc.

## Expected behavior

- `lineUp()` / `lineDown()` change `offset` by 1; `pageUp()` / `pageDown()` change it by `viewportHeight`.
- Offset clamps to `0 ... max(0, lineCount - viewportHeight)`. Movement past either end is a no-op (no wrap, no bell, no exit).
- At the bottom clamp, the state exposes an `atEnd` flag so the status bar can show `END`, but nothing terminates.
- Content shorter than one viewport makes all movement a no-op.

## Notes

Pure logic — implement as a plain observable class with no SwiftTUI imports and cover the clamp/boundary cases with unit tests (this needs a Tests target added to Package.swift; add it here). Consumes key events from [#0004](0004.md), rendered by [#0005](0005.md).

## Root cause

Greenfield: no scroll model existed. The model was built ahead of the key-event
plumbing (#0004) and view (#0005) because it is pure logic and independently
testable — the nominal "blocked by #0004" is only about wiring events into it.

## Fix

Added `PagerState` (Sources/NeoPager/PagerState.swift): a Combine
`ObservableObject` with `lines`, `viewportHeight`, and a clamped `offset`.
`lineUp/lineDown` step by 1, `pageUp/pageDown` by `viewportHeight`; all route
through a private `setOffset` that clamps to `0 ... max(0, lineCount - viewportHeight)`
and emits `objectWillChange` only on a real move (so a no-op at either end causes
no repaint). `atEnd`/`atTop`/`positionPercent` back the status bar;
`setViewportHeight` re-clamps on resize; `visibleLines()` returns the top-aligned
visible slice safely for any offset/viewport. No SwiftTUI import — SwiftTUI views
observe it through `ObservableObject`.

## Verification

`swift test` — 15 tests in `PagerStateTests` passed, 0 failures. Coverage:
start-at-top; line up/down with top/bottom clamping; "reaching the bottom is a
no-op, not an exit"; page movement by viewport with clamping; short-content
all-no-op + 100% position; position percentage at top/mid/end; resize re-clamping
(grow viewport pulls offset down, fit-all resets to 0); visible-slice correctness
near the bottom and with a zero viewport; and that `objectWillChange` fires on
real moves but not on clamped no-ops.

## Gotchas

The `defaultIsolation(MainActor.self)` package setting makes an `XCTestCase`
subclass MainActor-isolated, which conflicts with XCTest's `nonisolated` `init`
override ("main actor-isolated initializer has different actor isolation from
nonisolated overridden declaration"). The fix is two-fold: mark `PagerState`
`nonisolated` (it's pure logic, safe to call from any context), and give the test
target **no** `swiftSettings` so its `XCTestCase` subclasses stay nonisolated.
Apply this pattern to future test targets in this package.
