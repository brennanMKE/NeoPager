## Description

Phase 2: pressing `/` turns the status bar into a search input. Enter runs the search, jumps the viewport to the first match at or after the current position, and highlights all visible matches.

## Expected behavior

- `/` enters search mode: the status bar shows `/` and echoes typed characters; Backspace edits; Esc cancels and restores the normal status bar (without exiting the pager).
- Enter executes a case-insensitive literal substring search, scrolls so the first match at/after the current offset is visible, and stores the match list in `PagerState`.
- All matches in the visible slice are highlighted (e.g. inverse video on the matched range).
- No matches: the status bar reports `no matches for "query"` and the viewport doesn't move.

## Notes

Requires printable-character decoding from [#0004](0004.md)'s `KeyReader`. The double-duty Esc (cancel search vs exit pager) is the subtle bit — in search mode Esc only cancels search; the PRD specifies a second Esc then exits. Case-insensitive literal search first; regex can be a later ticket if ever wanted.

## Fix

`/` calls `PagerState.beginSearch()`, entering search-input mode (`searchInput`
becomes non-nil). The key handler is then modal: printable chars → `appendSearchChar`,
Backspace → `backspaceSearch` (empty cancels), Enter → `executeSearch`, Esc →
`cancelSearch`. This finally makes `q`/`b`/Space/etc. literal query characters while
typing — the gating caveat carried through #0013–#0020.

`executeSearch` runs a case-insensitive, non-overlapping literal substring search
(`String.range(of:options:.caseInsensitive)`) over every buffer line, storing
`[SearchMatch]` (buffer line + column range). It scrolls so the first match at/after
the current top buffer line is visible, or sets `searchStatus = no matches for "q"`
and leaves the viewport put.

Highlighting: each visible row is a `RenderRow` (windowed text + `startColumn`).
`PagerView.spans(for:)` intersects the line's matches with the row's column window and
splits the text into normal / match / current-match spans, rendered in an `HStack`
with distinct backgrounds (current = yellow on black, others = white on grey). The
status bar shows the `/query` prompt while typing, the `no matches` message, or the
`match N/M` count (#0011) once results exist. A first Esc after a search clears
highlights; a second Esc quits.

## Verification

`swift test` (67 total): 8 search cases — case-insensitive matches, non-overlapping
(`aa` in `aaaa` → starts 0,2), jump-to-first-at/after-top, no-match status + no
movement, `matches(onBufferLine:)` filtering, clear, and backspace/cancel. End-to-end
under a pty harness: `/NEEDLE`↵ echoes the prompt, jumps to the match, shows
`match 1/3`, and `NEEDLE` renders; the literal-character gating is proven because the
query "NEEDLE" searched correctly rather than its letters acting as commands; the
no-match message and Esc-clears-then-quits both work.

## Gotchas

Matches live in buffer-line/column space, independent of wrapping, so they survive a
re-wrap/resize; the view maps them onto rows via `startColumn`. Search uses
`range(of:options:)`, which needed `import Foundation` in `PagerState`. In chop mode a
match beyond the horizontal window isn't highlighted until you scroll to it
(`scrollToMatch` only adjusts the vertical offset) — fine for the default wrap mode.
Highlight composition with ANSI colors is #0012's concern. The exact highlight colors
are an eyes-on check for `closed`.
