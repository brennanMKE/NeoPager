# 0014 — Spacebar pages down, Shift-Space pages up

| | |
|---|---|
| **Status** | open |
| **Module** | Keys |
| **Platform** | macOS |
| **First seen** | 2026-06-10 |

## Description

Spacebar is the universal "next page" key in `less`/`more`/`man`, with Shift-Space as the conventional "previous page". NeoPager should support both: Space scrolls down one full viewport (same action as `⌥↓`/PageDown), Shift-Space scrolls up one full viewport (same as `⌥↑`/PageUp).

## Expected behavior

- A plain Space keypress pages down one viewport, clamping at the bottom (no exit).
- Shift-Space pages up one viewport, clamping at the top.
- Both are no-ops at their respective ends, consistent with the rest of the scroll model (#0006).

## Notes

**Space** arrives as the printable byte `0x20` → `RawKeyParser` emits `.char(" ")`. The pager's key handler routes `.char(" ")` to `PagerState.pageDown()` (the `.char` case was a no-op in phase 1).

**Shift-Space is terminal-dependent.** Most terminals (Terminal.app, default iTerm2, default Ghostty) send the *same* byte `0x20` for both Space and Shift-Space — they don't distinguish the two. A terminal only reports Shift-Space distinctly when an enhanced keyboard protocol is active:

- xterm `modifyOtherKeys`: `CSI 27 ; 2 ; 32 ~` (modifier 2 = shift, keycode 32 = space)
- kitty keyboard protocol: `CSI 32 ; 2 u`

`RawKeyParser` recognizes both of these forms and maps them to `.pageUp`. But we deliberately do **not** globally enable `modifyOtherKeys` or the kitty protocol, because both change how other keys are reported and would risk regressing the verified phase-1 input — notably, `modifyOtherKeys=2` can report Ctrl-C as a key sequence instead of raising `SIGINT`, breaking the Ctrl-C exit (#0008), and the kitty protocol changes bare-Esc reporting (#0008/#0004).

Consequence: Shift-Space pages up **only on terminals already configured to report it distinctly**; elsewhere Shift-Space behaves like Space (pages down). Reliable page-up remains `⌥↑` and PageUp. Opt-in enhanced-keyboard support (enable a protocol on startup, handle the re-encoded keys, restore on exit) can be filed as a follow-up if Shift-Space-everywhere is wanted.

The status-bar legend is left unchanged — it already advertises paging via `⌥↑/⌥↓`, and listing every page binding would crowd the bar.
