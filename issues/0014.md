## Description

Spacebar is the universal "next page" key in `less`/`more`/`man`, and `b` ("back") is its conventional counterpart for the previous page. NeoPager should support both: Space scrolls down one full viewport (same action as `‚å•‚Üì`/PageDown), `b` scrolls up one full viewport (same as `‚å•‚Üë`/PageUp).

## Expected behavior

- A plain Space keypress pages down one viewport, clamping at the bottom (no exit).
- `b` (and `B`) pages up one viewport, clamping at the top.
- Both are no-ops at their respective ends, consistent with the scroll model (#0006).

## Notes

Both keys are plain printable characters, so they arrive through `RawKeyParser` as `.char(" ")` and `.char("b")` ‚Äî no escape sequences, no terminal-dependent behavior, reliable everywhere. The pager's key handler routes them to `PagerState.pageDown()` / `pageUp()` (the `.char` case was a no-op in phase 1).

`b` was chosen over Shift-Space because most terminals send the same byte for Space and Shift-Space and can't distinguish them without an enhanced keyboard protocol that would risk regressing the verified phase-1 input (Ctrl-C/Esc handling). `b` is the `less` convention and Just Works.

Phase 2 caveat: once search input exists (#0010), `b`/Space are literal characters while the user is typing a query ‚Äî the paging bindings apply only in normal (non-search) mode. Same gating as `q`-to-quit (#0013).

The status-bar legend keeps advertising paging via `‚å•‚Üë/‚å•‚Üì`; Space/`b` are conventional aliases and listing every page binding would crowd the bar.

## Fix

In `NeoPager.run()`'s `keyHandler`, the `.char` case routes `" "` to
`PagerState.pageDown()` and `"b"`/`"B"` to `pageUp()`. Both arrive as plain
`.char` events from `RawKeyParser` ‚Äî no escape sequences, no parser change, no
enhanced-keyboard-protocol enable, so they work identically on every terminal.
Clamping/no-exit-at-bottom comes for free from the scroll model (#0006).

Shift-Space was dropped in favor of `b` (the original request was revised) because
most terminals send the same byte for Space and Shift-Space; `b` is the `less`
page-back convention and avoids that ambiguity entirely.

## Gotchas

Delta rendering makes end-to-end scroll assertions tricky: SwiftTUI rewrites only
the cells that changed, so after paging, line content like `line 78` never appears
as contiguous bytes (only the differing digit is written). Assert against the
status-bar **percentage** (which changes wholesale) or force a full repaint
instead of grepping for viewport line text.

Phase-2 caveat (shared with #0013): once search input lands (#0010), Space and `b`
are literal query characters while typing ‚Äî the paging bindings apply only in
normal mode.
