# 0014 — Spacebar pages down, b pages up

| | |
|---|---|

## Description

Spacebar is the universal "next page" key in `less`/`more`/`man`, and `b` ("back") is its conventional counterpart for the previous page. NeoPager should support both: Space scrolls down one full viewport (same action as `⌥↓`/PageDown), `b` scrolls up one full viewport (same as `⌥↑`/PageUp).

## Expected behavior

- A plain Space keypress pages down one viewport, clamping at the bottom (no exit).
- `b` (and `B`) pages up one viewport, clamping at the top.
- Both are no-ops at their respective ends, consistent with the scroll model (#0006).

## Notes

Both keys are plain printable characters, so they arrive through `RawKeyParser` as `.char(" ")` and `.char("b")` — no escape sequences, no terminal-dependent behavior, reliable everywhere. The pager's key handler routes them to `PagerState.pageDown()` / `pageUp()` (the `.char` case was a no-op in phase 1).

`b` was chosen over Shift-Space because most terminals send the same byte for Space and Shift-Space and can't distinguish them without an enhanced keyboard protocol that would risk regressing the verified phase-1 input (Ctrl-C/Esc handling). `b` is the `less` convention and Just Works.

Phase 2 caveat: once search input exists (#0010), `b`/Space are literal characters while the user is typing a query — the paging bindings apply only in normal (non-search) mode. Same gating as `q`-to-quit (#0013).

The status-bar legend keeps advertising paging via `⌥↑/⌥↓`; Space/`b` are conventional aliases and listing every page binding would crowd the bar.
