## Description

Esc is currently the single exit gesture (#0008), with Ctrl-C as the raw-mode safety net. `q` is the muscle-memory quit key for `less`/`more`/`man`, so users will reach for it. The pager should also exit when the user presses `q`.

## Expected behavior

- A plain `q` keypress exits the pager from any position, with the same clean teardown as Esc — terminal restored (cooked mode, cursor shown, alternate screen exited), exit code 0.
- The status-bar legend reflects the new binding (e.g. `Esc/q quit` instead of `Esc quit`), degrading gracefully on narrow terminals.

## Notes

`q` arrives as a printable character, so it surfaces through `RawKeyParser` as `.char("q")` — the pager's key handler already receives these (the `.char` case is currently a no-op in phase 1). Route `.char("q")` to the same `Application.quit()` path the `.escape` case uses.

Phase 2 caveat: once search input exists (#0010), `q` must **not** quit while the user is typing a search query — there `q` is a literal character. The exit binding applies only in normal (non-search) mode. Keep this in mind so the two features compose; if #0010 lands first, gate `q`-to-quit on not being in search-input mode.

## Fix

In `NeoPager.run()`'s `keyHandler`, the `.char` case now switches on the
character: `q` and `Q` call `app?.quit()` — the same `Application.quit()` teardown
Esc uses (#0008), so the terminal is restored and the process exits 0. `Q` is
included alongside `q` to match `less`. The status-bar legend changed from
`Esc quit` to `Esc/q quit` (both the full and short forms in `StatusBar`).

`q` needs no parser change: it arrives as `.char("q")`, already delivered to the
handler. Implemented together with the Space/`b` paging keys (#0014), which share
the same `.char` switch.

literal character while typing a query, not a quit. The `.char` switch will need
to be gated on not being in search-input mode. Currently there is no search mode,
so `q` quits unconditionally — correct for now, but #0010 must not forget this.
