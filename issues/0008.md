## Description

Esc is the single exit gesture. Quitting must restore the terminal to exactly its prior state — raw mode off, alternate screen buffer discarded so the user's scrollback is clean, cursor visible — and exit 0.

## Expected behavior

- A bare Esc keypress (as decoded by [#0004](0004.md)) terminates the pager from any position in the content.
- The terminal is restored: cooked mode, cursor shown, alternate screen exited (the paged content does not pollute scrollback).
- Ctrl-C behaves the same as Esc (restore, then exit) — a pager must never leave the terminal in raw mode on interrupt.
- Exit code 0 on Esc/Ctrl-C.

## Notes

Check what SwiftTUI's `Application` already handles for teardown (it installs the TUI; does it restore on SIGINT?). `Application.start()` calls `dispatchMain()` and never returns, so exiting likely means SwiftTUI's own exit path or `exit(0)` after explicit terminal restoration. Install a SIGINT handler that performs the same restore. Verify by running `git log | neopager`, exiting, and confirming `tput smso`/typed input behave normally and scrollback is intact.
