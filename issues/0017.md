## Description

`less` offers half-window scrolling — `d` forward, `u` backward (also Ctrl-D/Ctrl-U) — which is gentler for reading than a full-page jump. NeoPager has one-line and full-page movement (#0006) but nothing in between.

## Expected behavior

- `d` scrolls down by `floor(viewportHeight / 2)`; `u` scrolls up by the same amount.
- Clamps at both ends like all other movement; reaching the bottom is a no-op, never an exit.

## Notes

`d`/`u` are printable characters → `.char("d")` / `.char("u")` in the key handler → new `PagerState.halfPageDown()` / `halfPageUp()` methods.

Ctrl-D / Ctrl-U (bytes `0x04` / `0x15`) are currently swallowed by `RawKeyParser` (it skips C0 control bytes below `0x20`). Supporting them as aliases would need explicit parser handling — optional, since the plain letters cover the need. (Note `0x04` was the EOT/quit byte in upstream SwiftTUI's own input loop, but the pager's `keyHandler` path bypasses that.)

Phase-2 caveat (shared with #0013/#0014): `d`/`u` are literal query characters in search-input mode (#0010).

## Relation

- Pairs with: [#0006](0006.md)
- Same key-handler change as: [#0013](0013.md), [#0014](0014.md)

## Fix

`PagerState.halfPageDown()` / `halfPageUp()` move the offset by
`max(1, viewportHeight / 2)` (clamped, no-op at the ends — never exits). The key
handler routes `.char("d")` → `halfPageDown()` and `.char("u")` → `halfPageUp()`.
Ctrl-D/Ctrl-U were not added (they'd need parser support for the C0 bytes; the plain
letters cover the need, per the issue note).

## Verification

`swift test` (46 total): `PagerState` tests assert half-page moves by half the
viewport, clamps at `maxOffset`, and moves at least one row for a 1-row viewport.
End-to-end under a pty harness, an offset trace confirmed `d`,`d`,`u`,`u` from the top
produced offsets 19 → 38 → 19 → 0 (half of a 39-row viewport).

## Gotchas

Half-page uses `max(1, viewportHeight / 2)` so a one-row viewport still advances by a
line rather than zero. The status-bar percentage delta from a half-page move renders
non-contiguously (only changed cells are rewritten), so verify via an offset trace or
cumulative presses, not by grepping a single frame for the percentage.
