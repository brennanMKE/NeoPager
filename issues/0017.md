# 0017 — Half-page scroll: d / u

| | |
|---|---|
| **Status** | open |
| **Module** | Keys / Pager |
| **Platform** | macOS |
| **First seen** | 2026-06-10 |

## Description

`less` offers half-window scrolling — `d` forward, `u` backward (also Ctrl-D/Ctrl-U) — which is gentler for reading than a full-page jump. NeoPager has one-line and full-page movement (#0006) but nothing in between.

## Expected behavior

- `d` scrolls down by `floor(viewportHeight / 2)`; `u` scrolls up by the same amount.
- Clamps at both ends like all other movement; reaching the bottom is a no-op, never an exit.

## Notes

`d`/`u` are printable characters → `.char("d")` / `.char("u")` in the key handler → new `PagerState.halfPageDown()` / `halfPageUp()` methods.

Ctrl-D / Ctrl-U (bytes `0x04` / `0x15`) are currently swallowed by `RawKeyParser` (it skips C0 control bytes below `0x20`). Supporting them as aliases would need explicit parser handling — optional, since the plain letters cover the need. (Note `0x04` was the EOT/quit byte in upstream SwiftTUI's own input loop, but the pager's `keyHandler` path bypasses that.)

Phase-2 caveat (shared with #0013/#0014): `d`/`u` are literal query characters in search-input mode (#0010).
