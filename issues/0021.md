## Description

The test target uses XCTest (`import XCTest`, `XCTestCase` subclasses, `XCTAssertEqual`, …), which is the legacy testing system. All tests should use Swift Testing (`import Testing`, `@Test` functions, `#expect`/`#require`), the modern framework bundled with the Swift 6 toolchain.

## Expected behavior

- Every test file uses `import Testing`; none import `XCTest`.
- Tests are `@Test` functions (grouped into `@Suite` structs where helpful), using `#expect(...)` for assertions and `#require(...)` where a failed precondition should stop the test.
- No `XCTestCase` subclasses, no `XCTAssert*` calls remain.
- `swift test` discovers and runs all cases via the Swift Testing runner, all green, with no loss of coverage (the current 57 cases across `PagerStateTests`, `ContentSourceTests`, `RawKeyParserTests`).

## Notes

Swift Testing ships with the toolchain — test runs already report `Testing Library Version: 1902` alongside the XCTest suite, so no package dependency is needed.

Migration mechanics:
- `func testX()` → `@Test func x()`; `XCTAssertEqual(a, b)` → `#expect(a == b)`; `XCTAssertTrue(x)` → `#expect(x)`; `XCTAssertFalse(x)` → `#expect(!x)`.
- `XCTestCase` subclasses become plain structs (a fresh instance per test, like XCTest's setUp semantics); shared helpers (`makeLines`, `wrapLines`) become methods or free functions.
- Revisit the test-target isolation note from #0006: the target currently omits `defaultIsolation(MainActor.self)` to avoid `XCTestCase`'s nonisolated `init` conflict. Swift Testing has no such base class, so that constraint may no longer apply — but `PagerState`/`DisplayRow` are `nonisolated`, so the tests can stay nonisolated regardless. Confirm the chosen setting builds clean.

Once migrated, all **new** tests (e.g. for #0020, #0012, #0010, #0011) must be written in Swift Testing too.

## Fix

Converted all three test files to Swift Testing:
- `XCTestCase` subclasses → `@Suite struct`; `func testX()` → `@Test func x()`.
- `XCTAssertEqual(a, b)` → `#expect(a == b)`; `XCTAssertTrue/False` → `#expect(x)` /
  `#expect(!x)`. Failure messages carry over as the trailing `#expect(_, "msg")` comment.
- Shared helpers (`makeLines`, `wrapLines`, `parse`) stay as `private` instance methods —
  Swift Testing instantiates the suite struct fresh per test.
- `ContentSourceTests` gained `import Foundation` for `Data`, which XCTest used to
  re-export but Swift Testing does not.

The test target deliberately keeps **no** `defaultIsolation(MainActor.self)` (the
workaround from #0006): with XCTest gone there's no `XCTestCase.init` isolation conflict
to dodge, but `PagerState`/`DisplayRow` are `nonisolated` anyway, so the `@Test`
functions stay nonisolated and build clean either way.

## Verification

`swift test` — "Test run with 57 tests in 3 suites passed". `grep -rn 'XCTest'` over
`Tests/` returns nothing; no coverage was lost (same 57 cases).

## Gotchas

Swift Testing does not transitively import `Foundation` the way `import XCTest` did —
any test using `Data`, `URL`, etc. needs an explicit `import Foundation`. All **new**
tests (for #0020, #0012, #0010, #0011) must be written in Swift Testing.
