## Description

The defining feature versus `more`/`less`: a persistent one-row status bar at the bottom of the screen that always tells the user how to move and where they are. No memorized keys, no invisible state.

## Expected behavior

- A single row pinned to the bottom of the terminal, visually distinct from content (e.g. inverted/colored background).
- Left side: the key legend тАФ `тЖС/тЖУ line ┬╖ тМетЖС/тМетЖУ page ┬╖ Esc quit`.
- Right side: position тАФ percentage plus `line/total` (e.g. `37% 120/4882`); `END` when `PagerState.atEnd` is true, `100%` acceptable alternative.
- The bar never scrolls away and never overlaps content (viewport height already reserves this row, per [#0005](0005.md)).
- On narrow terminals the legend degrades gracefully (drop the position or shorten labels rather than wrapping to a second row).

## Notes

Build as a `StatusBar` SwiftTUI view driven by `PagerState`. In phase 2 the same row hosts the search input and match count ([#0010](0010.md), [#0011](0011.md)), so keep its layout componentized.

Greenfield: the defining feature versus `more`/`less` тАФ a persistent, self-documenting

## Fix

`StatusBar`, a one-row SwiftTUI view driven by `PagerState`: an `HStack` with the key
legend on the left, a `Spacer`, and the position on the right, framed to full width
and styled `.foregroundColor(.brightWhite).background(.blue)` so it's visually distinct
from content. The legend is `тЖС/тЖУ line ┬╖ тМетЖС/тМетЖУ page ┬╖ Esc quit`; the position is
`NN% bottomLine/total`, or `END` (replacing the percentage) when `PagerState.atEnd`.
Narrow-terminal degradation: the position is dropped when it won't fit, and the legend
falls back to a shorter form тАФ never wrapping to a second row (which would break the
viewport height math). Layout is componentized so phase 2 can host the search input /
match count in the same row (#0010/#0011).

## Verification

Under a controlling-pty harness, captured frames show the status bar rendered at the
bottom row with the styling escapes `\x1b[97m\x1b[44m` (bright white on blue) and the
live position тАФ e.g. after Down + PageDown + Option-Down the right side reads
`49% 118/200`, matching the exact offset math (39-row viewport: +1, +39, +39 тЖТ bottom
line 118 of 200, 49%). `END` and the percentage are also exercised by the `PagerState`
`positionPercent`/`atEnd` unit tests (#0006).

## Gotchas

`SwiftTUI` has no `.backgroundColor` text modifier; the row background is `.background(.blue)`
on a full-width frame. The position uses the *bottom* visible line number
(`min(offset + viewportHeight, lineCount)`), so it reads as "how far through" the file
you are. Visual confirmation of colors/contrast on a real terminal is the user's
eyes-on check for `closed`.
