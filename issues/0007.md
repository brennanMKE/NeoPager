## Description

The defining feature versus `more`/`less`: a persistent one-row status bar at the bottom of the screen that always tells the user how to move and where they are. No memorized keys, no invisible state.

## Expected behavior

- A single row pinned to the bottom of the terminal, visually distinct from content (e.g. inverted/colored background).
- Left side: the key legend — `↑/↓ line · ⌥↑/⌥↓ page · Esc quit`.
- Right side: position — percentage plus `line/total` (e.g. `37% 120/4882`); `END` when `PagerState.atEnd` is true, `100%` acceptable alternative.
- The bar never scrolls away and never overlaps content (viewport height already reserves this row, per [#0005](0005.md)).
- On narrow terminals the legend degrades gracefully (drop the position or shorten labels rather than wrapping to a second row).

## Notes

Build as a `StatusBar` SwiftTUI view driven by `PagerState`. In phase 2 the same row hosts the search input and match count ([#0010](0010.md), [#0011](0011.md)), so keep its layout componentized.
